hexo博客的搭建和使用

（hexo博客的搭建和使用)

我的作品集

起因是我想通过一个网页来展示自己在大学四年做过的一下课程设计小项目，进行一个归纳和总结。但因为网页的话需要租服务器，然后在查找的过程中发现了hexo这个基于Node.js的静态博客框架，依赖少易于安装使用，可以方便的生成静态网页托管在GitHub上，这样相当于自己的个人博客，因而采取了这个形式。在这里面放了一些以前的课程设计，现在接下来记录一下搭建hexo+github的过程。

hexo的优点在于
1.Node.js 所带来的超快生成速度，让上百个页面在几秒内瞬间完成渲染
2.支持markdown
3.只需一条指令即可部署到 GitHub Pages, 且指令短小易记
4.Hexo 拥有比较强大的插件系统（我只使用了本地图片自动上传这一插件，其他插件可以多加探索）

Hexo搭建过程

1.安装Git

Git是目前世界上最先进的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理，此处用来管理hexo博客文章。Windows版本的话直接去git官网下载，然后就会有一个git-bash.exe的命令行工具，相较cmd而言要好用很多。

2.安装Node.js

Hexo是基于nodeJS编写的，所以需要安装一下nodeJs和里面的npm工具。直接去官网下载即可。


安装完成后在 git-bash中输入 “node -v” 以及 “npm -v”来查看是否成功安装以及对应的版本号

SSH就是一个秘钥，其中，id_rsa是你这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。（来源：https://blog.csdn.net/sinat_37781304/article/details/82729029）
