网络游戏设计与开发课程设计

(网络游戏设计与开发—-坦克大战心得)

设计要求

网络游戏这个课程设计，老师当初的出发点是想让我们将完整的开发网络游戏的过程进行下来，所以根据教材将主要过程进行重新实现，然后进行分析学习。

单机部分

首先刚开始是要实现基本功能完整的单机游戏，实现坦克行走、旋转炮塔、发射炮弹、击杀提示、击杀音效、制造僵尸坦克、代码界面分离等功能

坦克控制单元

首先是引入坦克模型并添加碰撞体，然后为坦克添加控制代码
控制部分涉及到坦克的行走，旋转炮塔、发射炮弹等基本操作，接下来比较和之前不同的地方就是关于基于有限状态机的人工智能，有限状态机是指有限个状态以及在这些状态之间转移和动作的数学模型。如果不考虑数学模型，可以简单用ifelse语句来实现。


除此之外，还要设计代码分离的界面系统。
代码分离的界面系统，是指代码与界面资源的相互独立，将界面用代码封装起来，再通过一个管理器来控制，方便其他程序的调用和控制。如何实现代码分离的界面系统？
第一步：搭建界面，但不绑定自己写的逻辑脚本（基本库除外）
第二步：编写界面代码生成工具，根据界面上的名称和组件生成代码，包含：获取引用、添加事件侦听、移除事件侦听、事件函数等
第三步：动态绑定代码，在获取界面的时候动态绑定上指定的脚本即可（绑定组件的方式），写个公用函数会好一些。

房间部分

关于房间部分主要涉及的是协议的设计以及两个机制的应用
玩家登录游戏涉及到Login和Register协议，登出游戏将涉及到Logout协议。
打开房间列表后，面板左侧将显示玩家的成绩（总胜利次数和总失败次数），需要查询成绩的协议GetAchieve.
面板右侧将显示房间列表，须获得房间列表的协议GetRoomList，面板中有“新建房间”和“加入房间”按钮，将涉及CreateRoom和EnterRoom两条协议，面板中有加入房间，则需要获取（GetRoomInfo协议），玩家可以选择离开房间（LeaveRoom协议）或者开始战斗。
反射机制
程序通过“Msg”+name获取方法名methodName,然后使用GetMethod方法获取指定方法名的方法信息实例（MethodInfo)，得到了MethodInfo实例之后，通过Invoke就可以执行该方法，该方法原型如下：
Object Invoke (object obj,object[] parameters)
参数 说明
obj 一个对象的引用，将调用它所指向对象上的方法
parameters 所有需要传递给方法的参数都必须在parameters数组中指定

parameters 所有需要传递给方法的参数都必须在parameters数组中指定

委托机制
客户端消息分发机制使用到了委托。委托是一个类，它定义了方法的类型，使得可以将方法当作另一个方法的参数来进行传递，这种将方法动态地赋给参数的做法，可以避免在程序中大量使用if-else或者switch-case语句，同时使得程序具有更好的可扩展性。
委托是C#中的一种类型，它能够引用某种类型的方法，相当于C/C++中的函数指针，使用委托类需要注意以下几点：
1)声明一个delegata类型，它必须与要传递的方法具有相同的参数和返回值类型。
2)创建delegata对象，并将要传递的方法作为参数传入。
3)在适当的地方调用它。

心得

在本次的课程设计中，主要是完成一个坦克大战游戏的实现，从简单的单机游戏，再到房间系统的搭建，再到同步系统的完成，也将坦克大战这个游戏完善成一个可以多人对战的网络游戏。在搭建房间系统的过程中觉得最困难的是网络部分结构比较复杂，各个函数的调用关系很错综复杂。通过这次课程设计对游戏的创建过程有了直观的体验，而且课程的重点在于代码的理解，与上次的unity课设的侧重方向有所不同。
